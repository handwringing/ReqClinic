# 需求问诊室 · 人工浏览器验收脚本

> 文档状态：Next Productization Acceptance v0.1  
> 更新日期：2026-07-07  
> 适用范围：首页、快速问诊、正式项目、表达训练、报告页、反馈页、真实链路、示例体验、移动端和异常状态。

## 1. 使用规则

本脚本用于最终体验结论。结论必须来自 Browser 插件或同等真实浏览器操作，包括点击、输入、等待、刷新、返回和移动端视口检查；源码检查、接口调用和 DOM 断言只能作为定位手段，不能替代体验结论。

默认环境：

- 前端：`http://localhost:3000`
- 后端：`http://localhost:4200`
- 模型：后端本地配置的胜算云 `deepseek/deepseek-v4-flash`
- 移动端视口：`390 x 844`

失败分级：

- P0：白屏、崩溃、核心流程无法进入、报告/反馈无法生成、数据明显丢失。
- P1：用户不知道下一步、核心信息遮挡、真实链路和示例体验严重混淆、移动端核心流程不可用。
- P2：文案事实不符、等待反馈弱、状态不自然、报告层级影响理解、刷新恢复不稳定。
- P3：标题重复、留白、图标对齐、按钮质感、动效节奏、措辞精修。

禁用词检查：普通界面不得出现 `API Key`、`sample`、`custom`、`Mock`、`understanding_review`、`option_review`、`partial`、`inferred`、`missing`、`agent`、`skill`、`阶段 A` 等内部词。

## 3. 首页验收

入口 URL：`/`

操作步骤：

1. 首次打开首页，停留 5 秒。
2. 观察主品牌、主标题、副标题和三张入口卡片。
3. 分别点击快速问诊、正式项目、表达训练入口。
4. 从每个子页面点击“返回”，再点击品牌回首页。

预期页面表现：

- 用户能理解产品是“通过 AI 追问把需求说清楚”。
- 三个模式同级：快速问诊、正式项目、表达训练。
- CTA 清楚、可点击；返回只回上一个页面，品牌回首页。
- 页面不滥用英文，品牌可显示“需求问诊室 / Requirements Clinic”。

失败判定：

- 任一入口不可点击或进入错误页面为 P0。
- 三功能边界不清或标题重复影响理解为 P2。

Token：否。

## 4. 快速问诊真实链路

入口 URL：`/quick`

操作步骤：

1. 在输入框输入真实需求，例如“我想策划一场面向大学生的线下读书会，希望有人报名并能留下复盘材料”。
2. 点击开始问诊。
3. 回答 AI 的 2-3 个追问。
4. 进入当前理解确认，检查右侧整理内容是否逐步出现。
5. 点击可见卡片加入对话框，输入修改内容并发送。
6. 查看方案比较。
7. 生成简报，切换“普通概述 / 专业报告 / 导出文档”。
8. 刷新页面，再返回简报页。

预期页面表现：

- 真实输入可进入后端链路，不出现“填入回答”等示例专属按钮。
- 等待期间有进度反馈，按钮不能重复提交。
- AI 追问具体、自然，只问当前最有价值的问题。
- 右侧整理内容随对话逐步出现，信息量不过载。
- 卡片引用后输入框有清楚的一行引用状态。
- 方案比较至少两个方向，有差异化收益和代价。
- 三层报告同源，不出现 Markdown 原文、内部状态词或无依据事实。

失败判定：

- 真实链路不能到达简报为 P0。
- 无意义输入直接浪费模型或推进为完整需求为 P1/P2。
- 方案或报告明显编造未确认事实为 P1。

Token：是。

## 5. 快速问诊示例体验

入口 URL：`/quick`

操作步骤：

1. 点击“智能海报生成网站”等示例卡片。
2. 确认示例直接进入会话，不要求先手动填输入框。
3. 按提示点击“填入回答 / 发送”推进。
4. 在理解确认阶段点击指定右侧卡片加入对话框。
5. 点击“填入修改 / 发送”。
6. 查看方案、生成简报、切换三层报告。

预期页面表现：

- 示例体验速度应接近即时，不应表现为真实模型长等待。
- 示例按钮、文案、右侧卡片和报告闭环完整。
- 示例不会暗示其使用了用户输入或生成了真实项目。

失败判定：

- 示例需要长时间模型式加载为 P2。
- 示例流程卡在引用、修改、方案或简报为 P0/P1。

Token：否。

## 6. 正式项目：首页直接进入

入口 URL：`/formal/new`

操作步骤：

1. 空提交，观察错误提示。
2. 填写项目描述、相关人员、已有材料和项目约束。
3. 点击创建项目。
4. 等待需求地图生成。
5. 在工作台回答 1-2 个问题。
6. 点击地图节点加入对话框，修改节点内容并发送。
7. 打开报告，切换“普通概述 / 专业报告 / 导出文档”。
8. 刷新工作台，确认地图和对话恢复。

预期页面表现：

- 表单说明清楚：用户知道每个字段会如何用于地图。
- 创建后进入同一个地图工作台，不多一个无必要入口页。
- 左侧为对话，右侧为可扩展需求地图。
- 用户输入的人员、材料和约束在地图节点、对话追问或报告里可追踪。
- AI 不把未确认内容写成最终结论。

失败判定：

- 创建后崩溃、项目 ID 错误或地图空白为 P0。
- 表单输入后续完全没有用上为 P1。
- 报告只复述表单、没有专业结构为 P2。

Token：是。

## 7. 正式项目：快速问诊升级

入口 URL：快速问诊简报页 `/quick/{sessionId}/brief`

操作步骤：

1. 在快速问诊简报页点击升级正式项目。
2. 进入正式项目地图工作台。
3. 检查快速简报内容是否作为候选来源出现。
4. 回答正式项目追问。
5. 查看正式项目报告。

预期页面表现：

- 快速问诊结论只作为候选来源，不被当成正式事实。
- 工作台继续要求确认目标、范围、风险和交付标准。
- 报告显示版本来源，不隐藏待确认项。

失败判定：

- 直接把快速结论写成正式最终结论为 P1。
- 升级按钮使用登录、权限或开发中措辞阻塞为 P2，除非当前能力真实不可用并有自然说明。

Token：是。

## 8. 正式项目移动端

入口 URL：`/formal/{projectId}`，视口 `390 x 844`

操作步骤：

1. 进入正式项目工作台。
2. 检查顶部导航、对话区和移动端切换。
3. 依次切换“当前节点 / 地图总览 / 报告预览”。
4. 在当前节点加入对话并发送回答。
5. 打开报告 modal。

预期页面表现：

- 默认展示“当前节点”，用户先看到当前问题和最相关待确认内容。
- 地图总览支持 3-12 个节点滚动，不横向溢出。
- 报告预览有清楚入口，不需要用户在长地图里寻找。
- 输入框、节点按钮、报告 tab 不遮挡。

失败判定：

- 移动端横向溢出、节点菜单裁切、报告无法关闭为 P1/P2。

Token：视是否触发真实回答而定。

## 9. 表达训练完整闭环

入口 URL：`/training/cases`

操作步骤：

1. 从首页进入表达训练，选择固定情境。
2. 点击“填入追问”并发送。
3. 输入无意义内容 `123123`，检查提示。
4. 自定义输入一个有效追问并发送。
5. 把总结写入总结草稿，提交反馈。
6. 查看本轮覆盖、漏问维度、改写示例和学习轨迹。
7. 点击“同类再练一次”，确认创建同主题新练习。
8. 点击完成练习，返回情境页。

预期页面表现：

- 用户明确知道这是练习追问与总结，不影响真实项目。
- 角色回答自然，只回答已经问到的信息。
- 反馈说明“问到了什么、漏了什么、为什么重要、下次怎么问”。
- 学习轨迹只显示目标、对象、场景、边界、验收等用户可理解维度。

失败判定：

- 训练卡在等待、刷新后丢对话或反馈页空白为 P0/P1。
- 反馈泄露隐藏答案、答案 key 或规则为 P1。

Token：是。

## 10. 异常与边界

| 场景 | 操作 | 预期表现 | Token |
|---|---|---|---|
| 空输入 | 快速/正式空提交 | 留在当前页，自然提示需要补充什么 | 否 |
| 空格 | 输入空格后提交 | 同空输入 | 否 |
| 无意义输入 | 输入 `123123` 或 `asdfasdf` | 提示这句话不像需求/追问，要求补充，不直接推进 | 否或最小 |
| 过短但有意义 | 输入“做个海报” | 可进入问诊，由 AI 追问澄清 | 是 |
| 超长输入 | 粘贴长段说明 | 页面不崩溃，必要时提示精简或正常提交 | 是 |
| 重复点击 | 连续点击开始/发送/生成 | 第二次被禁用或明确等待 | 否/是 |
| 刷新恢复 | 刷新会话/项目/练习页 | 当前进度恢复，不回到死路 | 否 |
| 模型不可用 | 关闭后端模型配置后真实输入 | 提示模型服务暂未连接，并引导查看示例 | 否 |
