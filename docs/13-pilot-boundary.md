# 需求问诊室 · 小范围试用边界

> 文档状态：Pilot Boundary v0.1  
> 更新日期：2026-07-07  
> 目标：明确本地和小范围试用阶段的能力边界，不提前引入完整生产化复杂度。

## 1. 当前边界

当前版本用于本地演示、小范围试用和设计评审。它可以真实调用后端模型链路，生成问诊、需求地图、报告和训练反馈；但不承诺完整生产账号体系、多人协作、正式基线发布、审批流或长期数据治理。

暂不做：

- 登录注册和组织权限体系。
- 多人实时协作。
- 正式需求基线发布和审批流。
- SSE、工作流引擎、微服务拆分。
- 生产级数据清理后台。

## 2. 游客身份

本地和试用阶段允许使用受控游客身份进入快速问诊、正式项目首期和表达训练。

约束：

- 游客身份只用于演示和试用，不代表正式账号归属。
- 正式项目首期只生成候选地图和指导报告，不自动生成正式基线。
- 重要结论必须保留待确认项和版本来源。

## 3. 模型与 API Key

- DeepSeek 通过后端本地环境变量或本地启动脚本配置。
- 浏览器、前端代码、URL、日志和用户界面不得持有或展示 API key。
- 模型不可用时，真实输入应提示“模型服务暂未连接”，并引导使用示例体验。
- 示例体验不依赖模型，不应出现真实模型式长等待。

## 4. 数据保留与清理

试用阶段默认保留本地 SQLite 数据，便于复盘和审计。

建议策略：

- 试用前用独立数据库路径启动。
- 试用后按数据库文件整体备份或删除。
- 训练学习趋势仅保存在当前浏览器 localStorage，不进入项目资产。
- 导出文档来自当前 snapshot，不单独创建新事实。

## 5. 重复提交与长等待

所有真实模型链路必须满足：

- 提交后按钮立即进入等待态。
- 同一按钮等待期间禁用，防止重复提交。
- 长等待展示进度说明和当前正在做什么。
- 失败时给自然提示，并提供重试、返回或继续查看已有内容的路径。

## 6. 失败恢复

快速问诊：

- 会话页刷新后恢复对话、当前理解、方案或简报状态。
- 报告生成失败时保留当前理解和已回答内容。

正式项目：

- 工作台刷新后恢复项目、地图、对话和 active job 状态。
- 地图生成失败时显示可理解错误，允许返回项目起点或重试。

表达训练：

- 练习页刷新后恢复训练回合。
- 反馈生成失败时不丢失已问问题和总结草稿。

## 7. 试用准入

小范围试用前必须完成：

- `docs/11-manual-browser-acceptance.md` 的桌面和移动端主流程。
- 前端 `typecheck/build`。
- 后端 `typecheck/test`。
- 真实链路 token 审计导出。
- P0/P1/P2 清零；P3 可以记录在设计评审问题单里。
