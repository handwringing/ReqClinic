# 需求问诊室 / Requirements Clinic

需求问诊室是一个面向“说不清需求”的 AI 引导式需求分析产品。它不是让用户自己填写一长串表单，而是由 AI 连续追问，把模糊想法逐步整理成清楚的需求简报、需求地图和表达训练反馈。

## 产品定位

很多人在开始做一个项目、活动、论文、外包或协作任务时，并不是没有想法，而是不知道该先说明什么、哪些信息会影响结果、怎样表达才算完整。需求问诊室希望把这件事变成一个更自然的问诊过程：用户只需要回答当前问题，系统负责判断还缺什么，并把信息可视化整理出来。

## 核心功能

### 快速问诊

适合只有一句模糊想法的早期阶段。用户从一句话或示例体验开始，AI 会围绕目标、对象、场景、边界和完成标准连续追问，并逐步生成右侧可视化整理内容。最后输出普通概述和专业详细报告。

### 正式项目

适合需要进一步推进、协作或交付的需求。系统会把项目拆成可扩展的需求地图，围绕不同节点继续确认范围、角色、约束、风险和完成标准，帮助用户从“一个想法”过渡到“可以沟通和执行的项目说明”。

### 表达训练

适合练习需求沟通能力。用户选择固定情境，练习如何追问、如何总结，系统会给出反馈，指出已经问清的内容、遗漏的维度和更好的表达方式。训练结果与真实项目隔离，不会改写项目内容。

## 设计特点

- AI 主导澄清：用户不需要自己想“还该补什么”，只回答当前问题。
- 可视化整理：把长文本信息拆成卡片、地图节点和报告结构，降低理解成本。
- 普通话术与专业报告分层：概述给普通用户看，详细报告用于协作、评审和后续交付。
- 三种模式边界清楚：快速问诊解决早期澄清，正式项目承接深入分析，表达训练提升沟通能力。
- 示例体验可独立运行：即使没有连接模型服务，也能通过预设案例看到完整交互流程。

## 本地预览

仓库包含前端原型和后端服务。前端在未连接后端时会使用本地示例数据，适合直接体验产品界面和核心交互；连接后端后，可以进入真实的 AI 问诊链路。

```bash
cd ReqClinic
npm install
npm run dev
```

默认访问地址：`http://localhost:3000`。

如需体验真实模型链路，请在后端配置模型服务，并让前端连接本地后端。
